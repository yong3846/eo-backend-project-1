## 1. 회의한 내용

### 공통 CSS 수정 관련 논의

- **윤상이**
  - 공통 CSS를 사용하지 않고 개별적으로 적용하자는 의견 → X

- **현길용**
  - 공통 CSS를 조금 더 다듬어 작업해보자는 의견 → O

- **최서윤**
  - 헤더와 푸터 부분을 분리하여 공통 CSS를 적용하자는 의견 → O

### 로그인 로직 관련 논의

- 쿠키를 이해하고 적용하려 했으나,
  - 쿠키 값을 정상적으로 불러올 수 없는 문제 발생
- 대안으로 **localStorage 방식으로 전환하여 적용**하기로 함

### 전체 리뷰 관련

- 모든 작업 완료 후 전체 리뷰 및 수정 작업을 진행하기로 했으나
- 현재 작업이 미완성 상태이므로
  - 전체 리뷰는 다음 회의로 연기

## 8. 회고록

- **윤상이**
  - 쿠키에 대해 학습하고 이를 기반으로 구현을 시도했으나,
    쿠키 값을 불러올 수 없다는 한계를 확인함
  - 이후 다른 방식으로 구현을 전환했지만,
    쿠키를 사용해 적용하지 못한 점이 아쉬움으로 남음
  - 추후 쿠키를 불러오는 방식에 대해
    별도로 정리하고 학습할 필요성을 느낌

- **현길용**
  - 워크스페이스에는 최종 작업물만 두어야
    커밋 시 혼동을 줄일 수 있다는 점을 인식함
  - Pull Request는 신중하게 작성해야 하며,
    불필요한 기록이 남지 않도록 관리할 필요가 있음
  - 파일 이름 수정 또한 간단한 작업이 아니라는 점을 체감함

- **최서윤**
  - 각 페이지에 헤더와 푸터를 적용하고
    레이아웃 양식을 수정하는 과정에서
    HTML과 CSS 적용 방식을 복습할 수 있어 의미 있었음
  - 팀원들의 코드 작성 방식을 살펴보며
    다양한 구현 방식을 접할 수 있었고,
    전반적으로 유익한 시간이었다고 느꼈음
